Description:

Definition: Jdec Filter is a Filter which affects how final output is rendered.

There are 2  levels of filters in jdec:

  1. Class Level Filters
  2. Jar Level Filters(Archive to be precise)

    Class Level Filters:
   
    These filters as the name suggests has effect at the class level.
    These filters can sometime be useful to the user if the user for example
    does  not want the Synthentic or abstract methods to be decompiled.
    The above was only an example and can be extended to other cases.

    Please check the screenshot below for all class filters. Names are
    self explanatory.
 
   Jar Level Filters:
  •    These filters affect how jdec handles a jar file.For example the user   
           can choose to decompile only one package from the jar and NOT the
           entire jar. Jdec has support for this.Check the screenshot below.

  •    As of 1.2 release Jdec now supports archive files which are of
           zip format also. In other words if a zip file contains class files then       
           the user can just click on zip file.
  •    As of 1.2 release Jdec now allows user to define more archive level
      filters. User can select to decompile only class files belonging to default
          package Or any package also inside a archive file. It also allows the user
          to add/remove any other type of format to be recognized as archive file.
